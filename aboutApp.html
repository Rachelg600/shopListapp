About the shared list

Problem Statement

Family members have difficulty keeping track of items to buy within the household, resulting in items being duplicated or not being bought at all. Family members may ask each other to help buy stuffs or look out for certain items at an upcoming sale. Currently this is conveyed verbally or through Whatsapp. The person who is tasked to buy may forget about it after a few days or be stressed to hunt for the item(s) on the very same day.


Target audience

The target group for the solution will start with a core group of family members who stay in the same household. Family members staying together are more likely to be buying items for each other.


Proposed solution

Modelled after a to-do list, this is a simple web-app for family members to share and track their shopping list. Basic festures include:

  • Input field for item description
  • Drop-down list to select user; no creation of new user (yet) since this target audience is small
  • Entries are auto date-stamped
  • List displayed in a table that can be sorted (asc), e.g. to view items by user
  • "X"" button to allow user to remove an item; alert is displayed to confirm removal
  • Clear List button to allow user to clear entire list; alert is displayed to confirm removal
  • Alert to add item description when "+" button is clicked without entry

Future enhancement

The webapp could be improved in future:

  • Allow new users to be created
  • Allow user login (but this reduce useability with additional step)
  • Allow users to create multiple list, e.g. shopping list for upcoming overseas trip, shopping list for individual user

Challenges faced in this exercise

Struggling with backend, especially node and ejs, and database deployment, so app currently depends on local storage to store the list.
